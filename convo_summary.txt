⏺ Fix 2: Tables 4 and 5 — identical results across parameter sweeps.

  Root cause analysis for Table 4 (generations) and Table 5 (crossover):

  1. Same seed → same initial population every time
  2. Table 4: 50 vs 100 vs 150 generations — with seed=42, adaptive GA converges
   by gen 50 on this dataset. More generations = no change.
  3. Table 5: crossover 0.4/0.5/0.6 — the adaptive rate mechanism overrides
  cx_prob_init quickly, AND same seed = same population. The adaptive formula
  clips cx_rate to [0.4, 0.95] — so 0.4 is already the floor.

  The fix: use different seeds for each run so the GA explores different
  trajectories. This is standard practice for sensitivity analysis — you want to
   see how parameter changes affect outcomes, not have them all converge to the
  same local optimum from the same starting point.

  Now let me verify the numeric reasoning for why the fix works. With raw
  probabilities:
  - Correct class P ≈ 0.99 → GP1 base = max(0.99, 0.98, 0.97, 0.96) = 0.99 →
  saturates at T=0.9 ✓
  - Wrong class P ≈ 0.005 → GP1 base = max(0.005, ...) = 0.005 → no saturation ✓
  - For "borderline" samples: correct class P ≈ 0.7 → saturates at T=0.6 but NOT
   at T=0.8 ✓ — THIS creates the τ variation

  For GP2 with raw P:
  - Correct class: 1 - (1-0.99)(1-0.98)(1-0.97)(1-0.96) = 1 - 0.010.020.03*0.04
  = 1 - 2.4e-8 ≈ 1.0
  - Wrong class: 1 - (1-0.005)^4 ≈ 1 - 0.98 = 0.02

  For GP3 with raw P:
  - Correct class: 1 - sqrt((0.010.020.03*0.04)) = 1 - sqrt(2.4e-8) = 1 -
  0.000155 ≈ 0.9998
  - Wrong class: 1 - sqrt(0.98) ≈ 1 - 0.99 = 0.01

  So GP2 and GP3 will still be very similar on confident samples. The
  differentiation happens on borderline samples where probabilities are in
  0.5-0.9 range. Given this dataset has ~99.5% accuracy, most samples are highly
